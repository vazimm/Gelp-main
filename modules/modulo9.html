Módulo 9: Eventos e funções no JavaScript


Eventos


O que são eventos?


Eventos são ações ou ocorrências que acontecem na página web, geralmente iniciadas por um usuário ou pelo navegador. O JavaScript permite que você "escuta" esses eventos e executa um bloco de código (uma função) como resposta.

Exemplos de eventos comuns:

Interações do usuário:

  • Clicar em um botão (click)
  • Passar o mouse sobre um elemento (mouseover)
  • Retirar o mouse de um elemento (mouseout)
  • Pressionar uma tecla (keydown, keypress, keyup)
  • Mudar o valor de um campo de formulário (change)
  • Enviar um formulário (submit)

Eventos do navegador:

  • Carregamento completo da página ou de um recurso (load)
  • Redimensionar a janela (resize)
  • Rolagem da página (scroll)

Formas de Implementação de eventos


Existem várias maneiras de adicionar eventos em JavaScript. É crucial entender cada uma, mas também saber qual é a mais recomendada.


1. Diretamente no HTML (Atributos on*)


Esta é a forma mais antiga e não recomendada para a maioria dos casos, mas é útil para entender o conceito básico. O evento é adicionado como um atributo HTML, cujo valor é o código JavaScript a ser executado.

<button onclick="saudar()">Clique aqui!</button>
<input type="text" onchange="console.log(this.value)">

Vantagens: Simples para eventos muito pequenos e pontuais

Desvantagens: por que não é recomendado

  • Mistura HTML com JavaScript: Dificulta a manutenção e legibilidade do código. Idealmente, HTML deve cuidar da estrutura e JavaScript da lógica.
  • Difícil de gerenciar: Se você tiver muitos elementos ou precisar adicionar múltiplos event listeners ao mesmo elemento, o HTML fica poluído.
  • Escopo global implícito: O código dentro do atributo é executado no escopo global.
  • Remoção de eventos: É mais difícil remover ou modificar o evento dinamicamente.

2. Através do DOM com JavaScript (Proriedades on*)


Nesta abordagem, você seleciona o elemento HTML no JavaScript e atribui uma função à sua propriedade de evento correspondente (ex: onclick, onmouseover).


<button id="meuBotao">Clique em Mim!</button>
<script>
  const meuBotao = document.getElementById("meuBotao");

  // Atribui uma função ao evento click
  meuBotao.onclick = function() {
    alert("Você clicou no botão!");
  };
  // Você também pode atribuir uma função nomeada
  function lidarComHover() {
    console.log("Mouse sobre o botão!");
  }
  meuBotao.onmouseover = lidarComHover;

  // AVISO: Apenas uma função pode ser atribuída por evento.
  // Se você tentar adicionar outro:
  meuBotao.onclick = function() {
    console.log("Este será o único evento de clique!");
  };
  // A função anterior (alert) será sobrescrita!
</script>

Vantagens:

  • Separa o HTML do JavaScript.
  • Permite a manipulação de eventos dinamicamente.
  • Escopo local: O código dentro do evento é executado no escopo do elemento.
  • Remoção de eventos: Você pode facilmente remover ou modificar o evento dinamicamente.

Desvantagens: Só permite um event handler por evento: Como mostrado no exemplo, se você atribuir uma nova função ao onclick, a função anterior é substituída. Isso é uma grande limitação para aplicações complexas.


3. Usando addEventListener() (A forma mais recomendada)


O método addEventListener() é a maneira padrão e mais poderosa de registrar event handlers no DOM. Ele permite que você adicione múltiplos event listeners do mesmo tipo a um único elemento, sem sobrescrever os anteriores.


<button id="meuBotaoModerno">Clique Múltiplo</button>
<script>
  const meuBotaoModerno = document.getElementById("meuBotaoModerno");

  // Adiciona o primeiro event listener de clique
  meuBotaoModerno.addEventListener("click", function() {
    alert("Primeiro clique!");
  });

  // Adiciona o segundo event listener de clique (ambos serão executados)
  meuBotaoModerno.addEventListener("click", function() {
    console.log("Segundo clique!");
  });

  // Adiciona um event listener de mouseover
  meuBotaoModerno.addEventListener("mouseover", function() {
    this.style.backgroundColor = 'yellow'; });
  // 'this' refere-se ao elemento que disparou o evento

  // Para remover um event listener, você precisa da referência da função nomeada
  function handleMouseOut() {
    this.style.backgroundColor = '';
  }
  meuBotaoModerno.addEventListener("mouseout", handleMouseOut);

  // Para remover:
  //meuBotaoModerno.removeEventListener("mouseout", handleMouseOut);
</script>


Sintaxe de addEventListener():


elemento.addEventListener(evento, funcção, opção);

  • evento: Uma string que especifica o tipo de evento, ex: "click", "mouseover", "submit". (Note que não se usa "on")
  • funcao: A função a ser executada quando o evento ocorre.
  • opcoes (opcional): Um objeto que pode conter as seguintes propriedades:
    • capture: Um booleano que indica se o evento deve ser capturado na fase de captura (true) ou na fase de borbulhamento (false, padrão).
    • once: Um booleano que indica que o listener deve ser invocado no máximo uma vez. Se true, o listener é automaticamente removido após ser invocado.
    • passive: Um booleano que indica que a função listener nunca chamará preventDefault(), o que é útil para melhorar o desempenho de rolagem em dispositivos móveis.

Vantagens:

  • Permite múltiplos handlers: Várias funções podem responder ao mesmo evento em um elemento.
  • Separação de responsabilidades: Mantém o HTML limpo e focado na estrutura.
  • Controle sobre a fase do evento: Permite especificar se o handler deve ser ativado na fase de captura ou de borbulhamento.
  • Facilidade de remoção: O método removeEventListener() permite remover handlers específicos.
  • Flexibilidade: Pode ser usado para anexar eventos a qualquer objeto que suporte eventos (documento, janela, elementos do DOM).

4. Diferença entre addEventListener e Atributos HTML (on*) / Propriedades DOM (on*)


Característica Atributo HTML (onclick="...") Propriedade DOM (elemento.onclick = ...) addEventListener()
Localização No HTML No JavaScript No JavaScript
Número de Handlers Um por evento Um por evento (sobrescreve) Múltiplos por evento
Separação de Código Não (mistura HTML e JS) Sim (HTML e JS separados) Sim (HTML e JS separados)
Remoção de Evento Difícil (requer manipulação do atributo) Fácil (atribuir null) Fácil (com removeEventListener)
Fase de Evento Apenas Borbulhamento Apenas Borbulhamento Borbulhamento ou Captura
Recomendação Não recomendado Não recomendado para múltiplos handlers Altamente recomendado

Conclusão: Sempre prefira addEventListener() para lidar com eventos em JavaScript, a menos que você tenha um motivo muito específico e raro para não fazê-lo.


Funções


Criação de Funções com e sem Parâmetros


Funções são blocos de código reutilizáveis que podem ser executados sob demanda. Elas são essenciais para organizar o código e torná-lo modular.

Função Simples (sem parâmetros): Uma função sem parâmetros não precisa de informações externas para executar sua tarefa.

Exemplo:

// Declaração de uma função simples
function saudar() {
  alert("Olá! Bem-vindo(a) ao nosso site.");
}

// Como chamar a função
// saudar(); // Isso faria o alert aparecer imediatamente se fosse chamado no escopo global

// Exemplo com evento // Isso fará o alert aparecer quando o botão for clicado
document.getElementById("botaoSaudacao").addEventListener("click", saudar);

Função com Parâmetro(s): Funções com parâmetros permitem que você passe valores para dentro da função, tornando-a mais flexível e reutilizável.

Exemplo:

// Declaração de uma função com um parâmetro 'mensagem'
function exibirMensagem(mensagem) {
  console.log(mensagem);
  alert(mensagem);
}

// Declaração de uma função com múltiplos parâmetros
function somar(a, b) {
  let resultado = a + b;
  console.log("A soma é: " + resultado);
  return resultado; // Funções podem retornar valores
}

Escopo de Função


O escopo refere-se à acessibilidade das variáveis, funções e objetos em diferentes partes do seu código. Em JavaScript, o escopo é fundamental para evitar conflitos de nomes e gerenciar o ciclo de vida das variáveis.

Escopo Global
Variáveis declaradas fora de qualquer função são consideradas no escopo global. Elas podem ser acessadas de qualquer lugar no seu script, incluindo dentro de funções.


let nomeGlobal = "Alice"; // Variável no escopo global

function mostrarNomeGlobal() {
  console.log(nomeGlobal); // Acessa a variável global
}

mostrarNomeGlobal(); // Saída: "Alice"
console.log(nomeGlobal); // Saída: "Alice"

Escopo de Função (Local)
Variáveis declaradas dentro de uma função estão no escopo de função (ou local). Elas só podem ser acessadas de dentro da própria função onde foram declaradas.


function saudacaoComNome(nome) {
  let mensagemLocal = "Olá, " + nome + "!"; // Variável no escopo de função
  console.log(mensagemLocal);
}
saudacaoComNome("Bob"); // Saída: "Olá, Bob!"
// console.log(mensagemLocal);
// Erro: mensagemLocal is not defined (não pode ser acessada de fora da função)

Escopo de Bloco (let e const)
Com a introdução de let e const no ES6 (ECMAScript 2015), o JavaScript ganhou o conceito de escopo de bloco. Isso significa que variáveis declaradas com let ou const dentro de um bloco de código (delimitado por {}) só são acessíveis dentro daquele bloco. Variáveis declaradas com var não respeitam o escopo de bloco.


if (true) {
  let blocoLet = "Sou visível apenas dentro deste bloco 'if'";
  const blocoConst = "Também só aqui";
  var blocoVar = "Eu vazo para o escopo da função ou global"; // Cuidado com var!
  console.log(blocoLet);
}
// console.log(blocoLet); // Erro: blocoLet is not defined
// console.log(blocoConst); // Erro: blocoConst is not defined
console.log(blocoVar);
// Saída: "Eu vazo para o escopo da função ou global" (se estiver no escopo global ou de função)

Melhor Prática de Escopo:

  • Sempre use let ou const em vez de var para declarar variáveis. Isso ajuda a evitar problemas de escopo e torna o código mais previsível.
  • Mantenha as variáveis no escopo mais restrito possível (primeiro local, depois global se realmente necessário)
